# Foxbridge CDP Proxy

Foxbridge is a Chrome DevTools Protocol (CDP) to Juggler protocol translator. It lets tools that expect Chrome (like Puppeteer or raw CDP clients) control Camoufox/Firefox instead — transparently, without code changes.

**Repository:** [PopcornDev1/foxbridge](https://github.com/PopcornDev1/foxbridge)

**Status:** In progress — core domains mapped, gateway mode functional.

## Why Foxbridge?

OpenClaw and most browser automation tools speak CDP (Chrome DevTools Protocol) natively. Firefox uses Juggler, a different wire protocol. Without Foxbridge, you need the MCP bridge layer (see [MCP Tools](/mcp-tools)) to translate tool calls.

Foxbridge eliminates this indirection: it presents a CDP WebSocket endpoint that looks exactly like Chrome, but routes every command through Camoufox.

## Architecture

```
┌──────────────────┐     CDP WebSocket     ┌──────────────┐
│  OpenClaw /       │ ◄──────────────────► │  Foxbridge    │
│  Puppeteer /      │                      │               │
│  CDP Client       │                      │  Translates:  │
└──────────────────┘                      │  CDP → Juggler │
                                           │  Juggler → CDP │
                                           └───────┬───────┘
                                                   │ Pipe FD 3/4
                                           ┌───────▼───────┐
                                           │  Camoufox      │
                                           │  (Firefox)     │
                                           └───────────────┘
```

## Mapped CDP Domains

| CDP Domain | Juggler Equivalent | Status |
|-----------|-------------------|--------|
| `Page` | `Page.*` methods | Mapped |
| `Runtime` | Script evaluation | Mapped |
| `DOM` | Accessibility + DOM queries | Mapped |
| `Network` | Request interception | Mapped |
| `Input` | Mouse/keyboard dispatch | Mapped |
| `Target` | Context/page management | Partial |
| `Emulation` | Viewport/device | Planned |

## Usage

```bash
# Start Foxbridge as a CDP gateway
foxbridge --port 9222

# Connect with Puppeteer
const browser = await puppeteer.connect({
  browserWSEndpoint: 'ws://localhost:9222'
})
```

## Current vs Future Integration

**Current approach** (MCP bridge):
```
OpenClaw → MCP tools → vulpineos --mcp-server → Juggler → Camoufox
```

**Future approach** (Foxbridge):
```
OpenClaw → CDP WebSocket → Foxbridge → Juggler → Camoufox
```

Foxbridge removes one layer of translation and lets OpenClaw use its native CDP browser tools directly, rather than going through the MCP tool abstraction.

## Limitations

- Only core CDP domains are mapped (Page, Runtime, DOM, Network, Input)
- Some CDP-specific features (Service Workers, WebAudio profiling) have no Juggler equivalent
- Session management differs between CDP's flat target model and Juggler's context hierarchy
